生成式对抗网络(GAN)

2025-10-30
个人博客>学习笔记
4.6k 词
  • 深度学习

Network as Generator

把Network当作是个Generator来使用，原来输入的只有一个x；

现在除了x还要加上一个random variable $z$ ，这个$z$ 是通过一个简单分布(Simple Distribution)生成出来的，能知道表达式的分布都是简单分布(正态，均匀，伯努利……)。

比如说可以把$x$ ，$z$ 直接接起来，当作输入；也可以把$x$ ，$z$相加，变成一个输入。

每次输入的$z$都是从分布中随机生成的，生成的值是一个符合概率密度公式的随机结果。

可以输出一个分布(distribution)的Network，我们就叫它Generator。

为什么输出需要是一个分布?

如果在训练集里面，一个输入对应两个输出。

让机器的输出是有几率的，让机器不再是一个单一的输出，而是一个几率的分布。

比如说我们用二项分布，那么输出就有两种可能性。

我们想要让一个Function有同样的输入，但是有不同的输出，而且这些不同的输出都是对的

GAN(赣)

  • Unconditional generation

不输入x，只输入z，z是通过Distribution Simple出来的一个向量。

Discriminator（鉴别器）

鉴别器会拿一张图片作为输入，输出是一个数值，数值越大代表图片越正确。

Generator和Discriminator的实际算法

预先初始化生成器和鉴别器的参数。

Step1:固定生成器G的参数，只train鉴别器D的参数(可以当作分类问题，也可以当作回归问题)。

step2:固定鉴别器D的参数，只train生成器G的参数(让生成器去设法骗过鉴别器)。

实际的操作方法：生成器把一个向量(由简单分布产生)作为输入，然后产生一张图片；接下来我们把产生的图片丢到鉴别器D里面，鉴别器会给生成的图片一个分数；为了使输出的分数越来越高，我们改变生成器G的参数。

Theory behind GAN

在生成器里面，我们想要Minimize或者Maximize什么样的东西呢?

$PG$ 表示Vector经生成器之后产生的分布，$P{data}$是指真正的资料的分布。

我们期待$PG$的分布和$P{data}$的分布越接近越好。

Divergence between distributions $PG$ and $P{data}$。

在概率论/信息论里，divergence 指的是用来衡量两个概率分布 P 和 Q 有多“不同”的数量。

我们用散度来衡量$PG$ and $P{data}$之间的距离（因为他们之间的距离很难衡量）。

$G^*=argminGDiv(P_G,P{data})$

arg代表是一个参数，这表明要找到一组生成器G的参数来使$PG$ and $P{data}$之间的距离最小。

我觉得这个图可以这样理解，纵轴代表点在此x方向分布的概率。

但是Divergence用在这种连续的分布上面，需要做一个很复杂的，并且很难算出来的积分，根本就无法把Divergence算出来。

Sampling is good enough…….

GAN使用了很巧妙的方法解决。

我们不需要知道$PG$ 和 $P{data}$实际上的函式长什么样子，

只要我们能从这两个分布中Simple东西出来，我们就有办法算Divergence。

Discriminator

Discriminator训练的目标就是，看到real data给一个比较高的分数，看到generation data就给一个比较低的分数。

Training: $D^* = arg max_DV(G,D)$ V(G,D)里面的值和 JS divergence有关。

Objective Function for D：

​ $V(G,D) = E{y-P{data}}[logD(y)] + E{y-P{G}}[log(1-D(y))]$

我们希望 $V$ 越大越好，意味着我们希望$D(y)$越大越好。

事实上$D^$就相当于cross entropy乘上一个负号，所以在我们Max $D^$ 的时候，就相当于是在Min交叉熵，所以我们可以把Discriminator看作是一个classifier。

我们把真实的数据当作Class 1，把生成出来的数据当作Class 2，训练一个二分类的Classifier，训练完之后就相当于是找到了最优的参数。

为什么目标函数的值(我们反向传播用)和散度有关？

从直观上理解，假设$PG$ 和 $P{data}$的距离很小，那么相当于这两堆资料是混在一起的，所以很难分开，所以$D^$的值就不会很大；假设$PG$ 和 $P{data}$的距离很大，那么这两堆资料就可以轻易分开，所以$D^$就可以很大。

总结一句话：我们不知道Divergence怎么算，但是Classifier的交叉熵和Divergence是相关的，那么我们就训练一个Classifier。

G的目标是使红底框最小，D的目标是使红底框最大。

（好像高等数学那种巴拉巴拉极值问题）

Tips for GAN

GAN是以不好Train闻名的 NO PAIN BO GAN。

JS divergence其实并不合适

在大多数情况下，$PG$ 和 $P{data}$重叠的部分非常少，两种方式来说明。

1.nature of data: 图片其实是高维空间里面一个低维的manifold(流形？)，所以特定图片的分布在高维的空间中其实是非常狭窄的（数轴上几乎所有的点都不是有理数），只有非常小的范围才是图片。

2.Sampling：其实我们不知道$PG$ 和 $P{data}$的实际样子，我们对分布的理解来源于Simple，只要我们Simple的点不够多，就算是重合，我们也能找到一条线来使$PG$ 和 $P{data}$分开。

只要两个分布没有重叠，Divergence算出来的值就是$log2$。

使用JS在没有重叠的情况下，Classifier的ACC永远是100%，代表不了任何事情。

换一种衡量距离的方式：Wasserstein distance

想象是在开一台推土机，把$P$看成是一堆土，把$Q$看成是土堆放的目的地，从$P$到$Q$的平均距离就是Wasserstein distance。

穷举所有方法，寻找最小的平均距离，将其看作Wasserstein distance。

WGAN

把JS divergence换成Wasserstein distance就是WGAN。

D 必须是一个足够光滑的函数，这确保了只有generated和real距离很远的时候，它们两个之间的差距才会很大。（我们想让最大值最小）

如何限制函数的平滑性？

GAN仍然很难Train

生成器和鉴别器只要其中一个停止进步，另一个也会停下来。

GAN for Sequence Generation

TrainGAN最难的，其实是用GAN来生成文字。

文字的GAN和影像的GAN有什么不同呢，用梯度下降法去Train生成文字的Decoder，让鉴别器的分数变大，这件事做不到。

因为即使小小的改变了Decoder的参数，因为改变的参数很小，根本就影响不了one-hot的输出，或者说影响不了一个token，所以对鉴别器来说，输入还是一样的，从而输出也不变。

遇到不能用梯度下降法的问题，可以用Reinforcement Learning硬Train一发。

但是RL很难Train，GAN也很难Train，两个加在一起就会更加更加更加困难。

GAN for Sequence Generation

有人通过硬搜超参数和一堆Tips在不通过pretrain的情况下，也训练出来了Language GAN。

Possible Solution？

输入一个从简单分布随机出来的vector，然后每个vector都对应一张图片(label)；

然后用supervised learning(监督学习)的方法硬做。

能这样做，但是结果会很差，可能trian不起来。

GAN的评估

如何评估一个Generator的好坏？

一种做法是将图片放入一个影像分类系统，输出的结果是种类的概率分布，如果输出的概率分布非常集中，代表产生出来的图片比较接近真实的图片。

使用这个方法，评估的结果会被Mode Collapse骗过去。

Mode Collapse：在TrainGAN的时候，生成的图片只有固定的集中样式。

原因：某个地方是鉴别器的一个盲点，当生成器学会产生这种图片之后，就可以永远的骗过鉴别器。

解决办法（现在还没有太好的）: 在Mode Collapse之前，把Train停下来。

Mode Dropping：生成的样本缺乏多样性，只覆盖数据分布的一小部分。

判断多样性够不够：

Inception Score(IS): Good quality,large dicersity —— Large IS

借助Image Classifify，把生成的图片都丢到Classifify中，看其被判断为那种class。

每个图片都会有一个分布，我们把分布平均起来，如果得到的分布非常集中，就代表多样性不够；如果分布非常平坦，就代表多样性比较好。(适用于部分情况)

Frechet Inception Distance(FID):

先把产生出来的图片放到Inception中，用进入Softmax之前的Hidden Layer的输出，把这个向量代表这张图片。

假设产生出来的图片和真实的图片都符合高斯分布(正态分布)，然后去计算这两个高斯分布之间的Frechet Distance，距离越小越好。

使用高斯分布会有问题吗？ 会有问题。

如果我们需要准确的得到Network的分布，我们需要大量的Simple,这需要很大的运算量。

We don’t want memory GAN

我们不希望生成出来的图片和原来的图片一样。

所以评估一个GAN的Generate效果好不好是非常困难的。

Conditional GAN

我们给生成器一个x，让其根据x和z来产生y。

我们的鉴别器不止输入图片y，还要输入x；不仅图片要好，而且图片的叙述应该是和文字相配的。

我们还需要一种不好的情况：已经产生好的图片和文字叙述不相配。

我们也可以看一张图片而产生一张图片。(image translation,or pix2pix)

Learning from Unpaired Data

GAN怎么用在Unsupervised Learning上面？

什么时候会一点成对的资料都没有呢：影像风格转换

输入是X Domain的图片，输出是Y Domain的图片。

我们从X domain分布进行Simple，然后将其放到Generator里面，让其产生另一张图片。

我们再做一个Discriminator，合格的图片就给高分，不合格的图片就给低分。

但是这样的情况下，我们的输入和输出可能没有关系，输出只符合规范就够了。

Cycle GAN

在这里面，我们会Train两个Generator；

第一个Generator的工作是，把X Domain的图变成Y Domain的图；

第二个Generator的工作是，看到一个Y Domain的图，把它还原成X Domain的图。

我们做CycleGAN似乎没有办法保证，输入的图片和输出的图片看起来很像。

因为也许机器会学到一些很奇怪的转换。

这个问题没有什么很好的解法，但是实际上使用CycleGAN的时候，这样的状况没有那么容易出现。实际上，输入输出一般都比较像；但是在理论上没有保证。

同样的技术也可以用在Seq2Seq上。
